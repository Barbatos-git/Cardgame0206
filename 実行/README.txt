1．作品名
Card Game

2．コンセプト
本作品は、カードを「配置する」「重ねる」「組み合わせる」という
直感的な操作を通じて、プレイヤー自身が試行錯誤しながら結果を発見していく
涌現型カードゲームである。

固定された正解や攻略手順を用意するのではなく、
カードの種類・数・構成によって結果が変化する仕組みを採用し、
考える楽しさと発見する体験を重視している。

3．作品概要
本作品は、カードをドラッグ＆ドロップで操作し、
スタックや合成を行いながらゲームを進行させる
シングルプレイ向けカードゲームである。

プレイヤーはフィールド上にカードを自由に配置し、
合成エリアを利用して新たなカードを生成する。
合成結果は素材カードの組み合わせに依存し、
状況に応じた判断と工夫が求められる。

4．操作方法
【ゲーム中操作（マウス）】
・ドラッグ＆ドロップ：カードを移動
・カードスタック：同一種類のカードを重ねる
・カード合成：合成エリアに配置し、合成ボタンを押す
・マウスホイール：画面の拡大／縮小
・右クリック＋ドラッグ：画面の移動（カメラ移動）

【合成list】
・鉄鉱 + 樹脂 → 金属合金
・金属合金 + 石 → 構造鉄筋
・金属合金 + 木材 + 石→ 斧
・金属合金 + 木材 + 珪石 → つるはし
・金属合金 + 珪石 → 回路基板
・金属合金 + 生体外殻 → 強化複合シェル
・生体鉱 + フェロモン結晶 → 精製エネルギー触媒
・生体腺 + フェロモン結晶 → マイクロ脳層

5．主な実装内容・特徴
・涌現式合成システム
素材カードの種類や数、構成によって結果が変化する合成方式を採用。
固定レシピ方式も併存させ、拡張性を確保している。

・カードスタックシステム
単体カードとスタックカードを統一的に扱い、
安定した重ね・分離・移動処理を実現。

・合成エリア管理
複数の合成エリアを生成・配置可能。
合成ボタンは対応する合成エリアに追従。
展開／非表示は状態機械で管理。

・システム構成（asmdef）
Core / Data / Gameplay / UI / Editor の5モジュールに分離。
UI と Gameplay はインターフェース経由で通信し、疎結合を実現。

・静的レジストリによる管理
合成エリアは静的レジストリで管理され、
実行時探索に依存しない構造となっている。
